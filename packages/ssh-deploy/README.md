# sss-deploy-tools
使用SSH工具部署前端项目到远程windows server服务器, 或者部署前端项目到远程堡垒机的内部linux服务器(比如阿里云堡垒机环境)。

## 部署到堡垒机内部服务器(集群)大致流程

1. 检查本地目录是否存在
2. 打包项目(可选)
3. 连接到远程堡垒机
4. 检查堡垒机内部服务器是否存在待操作的临时目录和目标目录, 不存在时创建
5. 在远程服务器上备份项目文件
6. 堡垒机内部服务器临时目录下所有文件
7. 本地目录到堡垒机内部服务器临时目录
8. 删除堡垒机内部服务器目标目录下所有文件
9. 使健康检查失效
10. 将堡垒机内部服务器临时目录下所有文件移动到目标目录下
11. 使健康检查生效

## 部署到windows server服务器大致流程

1. 检查本地目录是否存在
2. 打包项目(可选)
3. 连接到远程服务器
4. 检查远程服务器是否存在待操作的临时目录和目标目录, 不存在时创建
5. 备份远程文件到本地
6. 删除远程服务器临时目录下所有文件
7. 上传本地目录到远程服务器临时目录
8. 删除远程服务器目标目录下所有文件
9. 将远程服务器临时目录下所有文件移动到目标目录下
